# NextChapter Agent Instructions

Before making any feature change to the application, read the active UI design
specification:

- `docs/superpowers/specs/2026-06-19-nextchapter-ui-design-spec.md`

Treat that document as the source of truth for the app's UI patterns, design
tokens, component conventions, accessibility expectations, and new-feature
checklist. If a requested feature appears to conflict with the spec, call out the
conflict and update the relevant SDD artifact before implementing the UI change.

Historical design notes may exist in this repo. They are context only unless an
SDD artifact explicitly marks them as active.
